Proposed phasing

All 8 features in one plan is too big to build safely. Here's a phased breakdown. We'd brainstorm and ship Phase A first, then come back for B, then C. Each phase delivers a usable editor — you're never stuck mid-feature.

PHASE A — FOUNDATION
Feel like Elementor
~3-4 weeks · the biggest unlock

What ships

  • Section → Column → Widget primitive
  • 20 widgets: Heading, Text, Image, Button, Icon, Icon Box, Divider, Spacer, Video, Accordion, Tabs, Counter, Progress Bar, Testimonial, Image Carousel, Gallery, Social Icons, Alert, HTML, Pricing Table
  • Inline WYSIWYG text editing (click + type)
  • Rich style panel: typography, color, bg (color/gradient/image), border, radius, 4-side padding/margin, shadow
  • Content / Style / Advanced tabs on every widget

Out of phase A: per-breakpoint values (preview only), template library, global styles, motion. Default values look good on mobile but you can't override yet.

PHASE B — PRODUCTIVITY
Build pages 10× faster
~2-3 weeks · after A is shipped

What ships

  • Per-breakpoint responsive values (Tablet/Mobile overrides for padding, font-size, visibility, etc.)
  • Template library: ~15 pre-built page templates (About, Pricing, Contact, Landing, Blog Post, etc.) + ~25 block templates (Hero variants, CTAs, etc.). Save your own as templates.
  • Global styles: 6 site colors + 4 typography presets (H1-H4, Body). Widgets reference these by default.

Requires: Phase A complete, since templates are just saved Section/Widget JSON.

PHASE C — POLISH
Make it sing
~2 weeks · after B is shipped

What ships

  • Entrance animations: fade, slide, zoom, on-scroll reveal
  • Hover states: hover styles + transitions on every widget
  • Advanced widgets: Form builder (with email submission), Google Map, Star Rating, custom Shortcode
  • Motion effects: simple parallax, sticky elements

Optional — the editor is fully usable without Phase C.
