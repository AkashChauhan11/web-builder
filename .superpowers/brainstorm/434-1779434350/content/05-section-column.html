Section & Column model

A few sub-decisions about how the primary layout primitive should work. Pick one from each group, or tell me in terminal.

A. How wide can a Section go?

A1 — Section has Boxed / Full-width toggle (Elementor)
Full-width section (edge to edge)
Boxed section (max 1200px)
Each section has a width property: Boxed (max-width + auto margins, content stays centered) or Full (100vw, bleeds to viewport edges — great for backgrounds).
A2 — Single max-width, page-level only
All sections boxed to 1200px
No per-section width control
Simpler. But you lose the ability to do dark full-width hero sections with boxed content. Less Elementor-like.

B. Column structures

B1 — Preset structures (Elementor)
When you add a Section, you pick from preset column layouts:

Drag column edges to adjust ratios afterward. Max 6 columns per section.
B2 — Free-form (add/remove columns)
You add a Section (single column by default), then click “+” to add another column. Drag column edges to size them.

C. Inner sections (nesting)

C1 — Yes, “Inner Section” widget (Elementor)
A column can contain an Inner Section, which itself has columns. Lets you build complex layouts (e.g. column with image on top + two columns of text below). Adds complexity but unlocks real Elementor-style designs.

C2 — No nesting (flat sections only)
Sections are flat siblings. Simpler model, easier to teach, fewer edge cases. You can still get most layouts by chaining sections, but complex split layouts inside one column become impossible.

My recommendation: A1 + B1 + C1 — the canonical Elementor model. Adds maybe 4-5 days to Phase A but it's why people pick Elementor in the first place. Doing this differently means you've built a different thing.
